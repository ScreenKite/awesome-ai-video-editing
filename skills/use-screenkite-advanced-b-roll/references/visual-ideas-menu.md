# Visual Ideas Menu

Beat-organized catalog for proposing B-roll visuals. Use as a starting point — adapt to what the speaker is actually saying.

## Density bundles

| Density | Count | Spacing | Feel |
|---|---|---|---|
| Sparse | 4 | ~40s | Documentary |
| Medium | 7 | ~24s | Default |
| Dense | 11–13 | ~13s | Explainer |
| Hyper | 15+ | phrase-level | Short-form |

**Default to Medium** for screen-recording tutorials.

## Ideas by beat

### Intro (0–10s)
1. **Lower-third nameplate** — host name + URL, 3.6s, bottomLeft
2. **Channel bug** — small persistent logo top corner
3. **Language badge** — "中文" or "EN", tiny corner chip

### Tool / product name reveal
4. **Big title card** — product name display type, Chinese subtitle, date stamp
5. **Logo stamp** — product logo with sparkle animation
6. **"Just launched" ribbon** — diagonal date banner
7. **News-ticker** — horizontal marquee of launch-day headlines

### Analogy / comparison
8. **Side-by-side cards** — competitor vs. this product with feature chips
9. **Feature trinity chips** — 3 labeled pills stacked or in a row
10. **Evolution diagram** — arrow flow showing what changed
11. **Speed line** — "from X to Y" with progress bar

### Technical concept (connector, flow, data)
12. **Connector diagram** — N input nodes → arrow → output node
13. **Architecture badge** — small icon + tech label (Chrome API, GitHub, etc.)
14. **Privacy/safety callout** — shield icon + reassurance text
15. **Before/after panel** — left old way, right new way

### Loading / wait state
16. **Loading clock** — stylized timer, not literal
17. **Progress bar** — filling bar with token/time meter
18. **"Please wait" card** — tone-matched casual card

### Feature recap
19. **Feature grid** — 3×2 icons with labels
20. **Quota/plan split** — stacked bars showing separate resources
21. **Highlight sticker** — emoji-backed emphasis ("rare!", "generous!")
22. **Price tag pop** — animated price callout

### Self-promo / plug
23. **App icon + tagline** — product card with orbit animation
24. **vs. competitor checklist** — feature parity table
25. **Meta-moment card** — "this video made with X" callout
26. **CTA arrow** — "link below ↓" with pulsing chevron
27. **Roadmap timeline** — Now → Soon → Future checkpoints

### Tool-use / process
28. **Node graph** — center node + orbiting satellites
29. **Analogy card** — "like hiring a designer" avatar + text
30. **Terminal log** — fake tool-call log (→ read file X)
31. **Cost meter** — coin stack or $ incrementing

### Outro
32. **End card** — "see you in X" with subscribe bug
33. **Teaser** — "Part 2 →" or "next video"

## Proposal format

When presenting the menu, structure like:

```
Beat A — Intro (0–10s)
  1. Lower-third nameplate
  2. Channel bug

Beat B — Tool name (10–25s)
  4. Big title card
  5. Logo stamp
...
```

Then offer three bundles the user can pick from (e.g. "Bundle M7 = 1, 4, 8, 12, 20, 23, 28"), and ask for their pick or a custom mix.

**Wait for confirmation before scaffolding.** Generating visuals the user doesn't want wastes ~10min of render time per unwanted slot.
